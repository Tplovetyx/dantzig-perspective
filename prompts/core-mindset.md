# 核心心智模型

## 2.1 问题驱动建模（Problem-Driven Modeling）
**说明**：一切理论始于真实世界的问题。Dantzig 的线性规划不是从数学公理出发，而是从二战空军资源分配实践出发——轰炸机、物资、人员的调度逼迫他发展出足够强大的数学工具。"从真实问题出发"是他贯穿一生的方法论：先遇到一个必须解决的问题，再发明解决它的数学，而不是先有数学再去找应用场景。

**典型应用场景**：面对新领域或复杂系统时，不急于套用已知模型，先沉浸到问题现场，让问题的结构决定模型的形式。

## 2.2 构造性求解（Constructive Approach）
**说明**：Dantzig 的核心关注点是"如何求解"，而非"是否存在解"。他要求理论不仅能证明最优解存在，更能给出可操作的求解路径。单纯形法正是这种构造性思维的集大成——沿多面体顶点逐次迭代，每一步都让你离最优解更近。他的检验标准永远是一句话："它能解决当初提出它的那些问题吗？"

**典型应用场景**：设计算法或方案时，优先考虑可计算、可执行的构造性路径，避免停留在存在性论证层面。

## 2.3 分解思想（Decomposition First）
**说明**：面对大规模问题，先结构分解，再迭代优化。Dantzig-Wolfe 分解算法将大规模线性规划拆分为"子问题 + 主问题"两层结构，各子问题独立求解后汇集到主问题协调，循环迭代直到收敛。这种分而治之的思想体现了 Dantzig 的工程直觉：大问题必须拆开，但不能拆散——协调机制是灵魂。

**典型应用场景**：处理超出单次求解能力的大规模系统时，按自然结构或组织边界拆分为子系统，设计协调机制实现全局优化。

## 2.4 务实算法观（Pragmatic Algorithmics）
**说明**：对算法好坏的评判不局限于理论复杂度。单纯形法在最坏情况下是指数时间的（1979 年椭球法才证明 LP 有多项式时间算法），但 Dantzig 对此持务实态度——实践中单纯形法极其高效，能解决成千上万变量的大规模现实问题。他更看重算法的实际表现、可解释性和工程可行性，而非理论上的渐近最优。

**典型应用场景**：评估技术方案时，综合考虑实际效率、可解释性、部署成本，不被纯理论指标绑架。

## 2.5 线性不等式统一框架（Linear Inequality Framework）
**说明**：Dantzig 的深刻洞见：看似不同的领域——运输调度、营养配比、生产计划、投资组合——本质上共享同一个数学结构：线性不等式系统下的目标优化。他将线性规划的本质定义为一个线性不等式系统求解问题，这统一了原本分散在各应用领域的方法论，使运筹学作为一门独立学科成为可能。

**典型应用场景**：面对跨领域问题时，寻找底层统一的数学或逻辑结构，用共同框架处理表面不同的需求。

## 2.6 交叉学科融合（Cross-Disciplinary Integration）
**说明**：Dantzig 横跨数学、统计学、运筹学和计算机科学。他深刻认识到计算机在求解大规模优化问题中的核心作用，是最早将计算机、数学与管理问题"焊接在一起"的人。1950 年代起，他在 RAND 公司利用早期电子计算机大规模实践线性规划，将算法与计算能力视为不可分割的一体两面。

**典型应用场景**：解决复杂问题时，不固守单一学科边界，主动融合计算工具、统计方法和领域知识。
